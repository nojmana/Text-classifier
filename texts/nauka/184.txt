Microsoft od dawna jest świadom istotności deweloperów dla swojego rozwoju oraz tego, jak ważne jest mieć potężną społeczność programistów po swojej stronie. Przejmując GitHub firma wzbogaca się właśnie o taką społeczność – 27 milionów programistów, którzy używają repozytorium jako krytycznej części swoich procesów wytwórczych oprogramowania. Również programiści reprezentują kluczowy rynek dla technologii chmurowych Microsoft.Zakup GitHub to drugie duże przejęcie Microsoftu za kadencji Satya Nadella, po przejęciu za 26,2 mld USD LinkedIn dwa lata temu. GitHub był wyceniony na 2 mld USD w 2015 roku, ale Microsoft płaci 7,5 mld USD za pełen pakiet akcji firmy, a transakcja powinna zamknąć się jeszcze w tym roku.Pytanie jakie rodzi się wraz z tym przejęciem to: co Microsoft zrobi z GitHubem w przyszłości. LinkedIn w dużej mierze pozostał niezależny, z niektórymi funkcjami zintegrowanymi z Microsoft Office. Przejęcie Minecraft przez Microsoft jest oceniane równie dobrze i z dużym prawdopodobieństwem GitHub będzie musiał pozostać jak najbardziej autonomiczny, aby utrzymać zaufanie deweloperów. Możemy jednak sobie wyobrazić dalej idącą integrację z narzędziami rozwoju oprogramowania Microsoftu. W ubiegłym miesiącu firma ogłosiła, że w ramach swojej bliskiej współpracy z GitHub integruje usługę z App Center. Użytkownicy będą się też bacznie przyglądać, czy i w jaki sposób będzie integrowany z chmurą Azure.Pomimo wszystkich obaw istnieją przesłanki, aby uznać przejęcie za pozytywne tak dla użytkowników Microsoft jak i GitHub. Microsoft aktywnie angażuje się w Open Source, a firma upubliczniła PowerShell, Visual Studio Code i silnik JavaScript Microsoft Edge. Microsoft nawiązał także współpracę z Canonical wprowadzając system Ubuntu dla Windows 10. Następnie przejął Xamarin, aby zaistnieć na polu tworzenia aplikacji mobilnych. Są to posunięcia, które początkowo zaskoczyły deweloperów, ale w dalszej kolejności zyskały aprobatę. Ważne będzie jak firma podejdzie do społeczności GitHub i czy uda się zdobyć jeszcze więcej zaufania.
